# 论文深度剖析（MCP 工具版）

用户在任何支持 MCP 的客户端（Claude Desktop、Cursor、你自己的 OpenCode 等）里就能直接调用。你的 prompt 库从"手动复制粘贴"升级为"即插即用的工具"。

## 什么时候才需要 Agent

如果你的目标变成这样，才需要 Agent：

"我扔一篇论文进去，它自动：先翻译、再剖析核心方法、然后生成知识卡片、最后出一份学习笔记——中间还能根据我的理解程度调整深度"

这是一个多步编排 + 状态管理 + 动态决策的场景。但说实话，这完全可以在 MCP 的基础上再加一层轻量编排（比如用一个 Agent prompt 来串联多个 MCP tool），而不需要从零搭一个 Agent 框架。

## 我的建议路径

- 先做 MCP：把你的 prompt 库暴露为 MCP tools -> 立即可用，开发快
- 再做编排层（如果需要）：写一个"论文学习 Agent" prompt，它的工具就是你的 MCP tools -> 实现多步自动化
- 不要一开始就做 Agent 框架：过度工程，你的内容不需要那么重的架构
